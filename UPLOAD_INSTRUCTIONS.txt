ИНСТРУКЦИЯ ПО ЗАГРУЗКЕ САЙТА НА GITHUB (самый простой способ)

Текущая дата: 2 июля 2026

=== ЧТО ДЕЛАТЬ ПРЯМО СЕЙЧАС ===

1. Открой репозиторий:
   https://github.com/2012leon27-oss/usenko-masterclass

2. Нажми большую зелёную кнопку "Add file" (вверху справа) → "Upload files"

3. Открой на своём компьютере эту папку:
   G:\сайт для первого запуска андрея усенко\docs

   Внутри ты увидишь:
   - index.html
   - папку static
   - файл .nojekyll (может быть скрыт)

4. ПЕРЕТАЩИ ПАПКУ "docs" ЦЕЛИКОМ в белую область на сайте GitHub 
   (или нажми "choose your files" и выбери папку docs).

   После этого у тебя в репозитории появится папка "docs".

5. Внизу страницы в поле "Commit changes" напиши:
   Add landing page

   И нажми зелёную кнопку "Commit changes" (или "Commit directly to the main branch").

6. Подожди 5-10 секунд, пока файлы загрузятся.

=== ВКЛЮЧАЕМ GITHUB PAGES ===

7. Вверху репозитория перейди во вкладку **Settings** (Настройки).

8. В левом меню найди и кликни **Pages** (Страницы).

9. В разделе "Build and deployment" найди "Source":

   - Выбери: Deploy from a branch
   - Branch: main   (или master, если вдруг так)
   - Folder: /docs     ← обязательно выбери /docs !

10. Нажми синюю кнопку **Save**.

11. Подожди 30–90 секунд.

=== ГОТОВО ===

GitHub покажет надпись примерно такую:

"Your site is published at 
https://2012leon27-oss.github.io/usenko-masterclass/"

Это и есть твоя постоянная ссылка.

Открой её в браузере.

Если сайт не появился сразу — подожди ещё минуту и обнови страницу.

=== ЧТО ДЕЛАТЬ ПОТОМ ===

- Когда сайт откроется, пришли мне ссылку, которую ты получил.
- Если нужно будет поменять ссылки на кнопках — скинь мне их, я обновлю файл и ты просто зальёшь заново.

Удачи! Если на каком-то шаге застрянешь — пиши сюда, что именно видишь.
