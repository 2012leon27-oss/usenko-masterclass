# Сайт мастер-класса Андрея Усенко

Одностраничный лендинг для бесплатного 2-дневного мастер-класса (8-9 июля).

## Быстрый предпросмотр (без установки пакетов)

```bash
python serve.py
```
Открой http://localhost:8000 — это точная копия макета, которую можно открыть прямо сейчас.

## Важно: Локальный запуск НЕ работает после выключения компьютера

`python app.py` и `python serve.py` работают **только пока твой компьютер включён**.

Чтобы сайт был доступен людям постоянно (даже когда ты выключил ПК) — нужно его опубликовать на внешнем хостинге.

Самый простой вариант именно "на GitHub" — **GitHub Pages** (статический хостинг).  
Подробно расписано в файле **GITHUB_PAGES.md**.

## Как запускать локально (только для теста)

1. Установи зависимости:
```powershell
pip install -r requirements.txt
```

2. Самый простой способ (статическая копия, без Flask):
```powershell
python serve.py
```
Открой http://localhost:8000

3. Полноценный Flask (покажет предупреждение — это нормально):
```powershell
python app.py
```
Открой http://localhost:5000 или http://192.168.31.77:5000 (твой локальный IP)

**На Windows для более "продакшен-подобного" запуска** можно использовать waitress:
```powershell
pip install waitress
waitress-serve --listen=0.0.0.0:5000 app:app
```

Предупреждение, которое ты увидел:
> WARNING: This is a development server. Do not use it in a production deployment.

Это Flask честно говорит, что встроенный сервер не для продакшена. В проде будет использоваться gunicorn (на хостинге).

## Развёртывание (чтобы сайт работал всегда, даже когда компьютер выключен)

**Самый простой и быстрый вариант именно на GitHub — GitHub Pages.**

Я уже подготовил для этого папку `docs/`.  
Подробная инструкция здесь: [GITHUB_PAGES.md](./GITHUB_PAGES.md)

GitHub Pages даёт постоянную бесплатную ссылку вида:
`https://твойлогин.github.io/название-репозитория/`

Это полностью статическая версия. Python на сервере не запускается, но для лендинга с кнопками этого более чем достаточно.

### Другие варианты (если хочешь именно Python на сервере)

### Вариант A — Render.com (было рекомендовано раньше)

1. Зарегистрируйся на https://render.com
2. New Web Service → Connect GitHub (или используй "Deploy from local" / CLI если нужно).
3. Выбери этот репозиторий.
4. Render автоматически подхватит `render.yaml`.
5. После деплоя получишь публичную ссылку вида `https://andrey-usenko-masterclass.onrender.com`

Примечание: на бесплатном тарифе сайт "засыпает" после 15 минут бездействия. Первый визит может быть 30-60 сек. Для всегда-онлайн нужен paid план ($7/мес).

### Вариант B — Railway.app

1. railway.app → New Project → Deploy from GitHub.
2. Build settings: `pip install -r requirements.txt`
3. Start: `gunicorn app:app --workers 2 --threads 4 --timeout 60`

### Вариант C — PythonAnywhere (хорошо для простых Flask)

- Загрузи файлы через Files.
- В Web tab создай Flask app, укажи `app.py` + `app` как flask_app.
- На бесплатном всегда-онлайн ограничен, но можно.
