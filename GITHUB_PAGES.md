# Развёртывание на GitHub Pages (бесплатно и всегда работает)

Это самый простой способ разместить сайт на GitHub, чтобы он работал 24/7 даже когда твой компьютер выключен.

**Важно:** GitHub Pages — это только для статических сайтов (HTML + CSS + JS). Python (Flask) на нём не запускается.  
Поскольку твой лендинг — это просто красивая страница с кнопками-ссылками, этого достаточно.

## Быстрый способ (рекомендую)

### Шаг 1: Подготовка (уже сделано)
В папке проекта уже есть папка `docs/` с готовой статической версией сайта.

### Шаг 2: Создай репозиторий на GitHub
1. Перейди на https://github.com/new
2. Назови репозиторий, например: `usenko-masterclass` или `andrey-landing`
3. Сделай его **Public**
4. НЕ инициализируй README, .gitignore и лицензию (мы зальём свои файлы)

### Шаг 3: Загрузи файлы в GitHub

**Вариант A — через Git (если умеешь)**

```powershell
cd "g:\сайт для первого запуска андрея усенко"

# Инициализируем git
git init
git add .
git commit -m "first version of landing"

# Подключи свой репозиторий (замени ссылку)
git remote add origin https://github.com/ТВОЙ_ЛОГИН/usenko-masterclass.git

# Пушим на главную ветку
git branch -M main
git push -u origin main
```

**Вариант B — просто через веб-интерфейс GitHub (без командной строки)**

1. Открой свой новый репозиторий на GitHub
2. Нажми кнопку "Add file" → "Upload files"
3. Перетащи папку `docs` целиком
4. Также загрузи корневые файлы, если хочешь (README, serve.py и т.д.)
5. Или просто загрузи всё содержимое папки `docs` в корень репозитория (тогда не нужно будет настраивать папку)

### Шаг 4: Включи GitHub Pages

1. Зайди в свой репозиторий на GitHub
2. Перейди во вкладку **Settings** → **Pages** (слева в меню)
3. В разделе "Source" выбери:
   - **Deploy from a branch**
   - Branch: `main`
   - Folder: `/docs`   ← важно!
4. Нажми Save
5. Подожди 30–60 секунд

Готово! GitHub покажет зелёную ссылку примерно такого вида:

`https://твойлогин.github.io/usenko-masterclass/`

Это и будет твоя постоянная публичная ссылка.

## Альтернативный простой вариант (ещё быстрее)

Если не хочешь возиться с папкой `/docs`:

1. Создай в репозитории отдельную ветку `gh-pages`
2. Загрузи в неё **только содержимое** папки `docs/` (index.html + папку static)
3. В Settings → Pages выбери ветку `gh-pages` и корневую папку `/ (root)`

## После деплоя

- Сайт будет работать всегда.
- Можно будет давать людям прямую ссылку.
- При желании позже подключишь свой домен (в настройках Pages).

## Важно про ссылки на кнопки

Сейчас кнопки всё ещё ведут на `#`.  
Как только скинешь мне реальные ссылки — я обновлю `docs/index.html` (и при необходимости другие файлы), и ты просто зальёшь новую версию.

## Если потом понадобится настоящий Python-бэкенд

GitHub Pages не умеет запускать Python.  
Тогда придётся использовать:
- Railway
- Render
- Fly.io
- PythonAnywhere

Но для текущей задачи (одна посадочная страница) — GitHub Pages идеально.
